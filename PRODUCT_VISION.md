## 1. 产品使命（Mission）

玄风门存在的意义，不是为用户排一次八字。

我们相信，命理的价值在于帮助人们在面临重要人生决策时，获得更可信、更容易理解、更具有行动价值的参考。

不是预言。
不是替代思考。
而是辅助理解、辅助分析、辅助决策。

> 让每个人都能在人生关键时刻，获得值得信赖的命理视角。

---

## 2. 产品愿景（Vision）

未来五年，玄风门希望成为中文世界最值得信赖的数字命理平台。

这里的关键词不是：最准、最神、最玄。

而是：

| 关键词 | 含义 |
|--------|------|
| **可信** | 每个结论都有来源，每个结果都可以追溯 |
| **透明** | 算法逻辑公开可验证，不藏黑箱 |
| **长期主义** | 不追求短期流量，追求持续积累和用户长期信任 |

我们不追求"命中一切"的神秘感。
我们追求的是"有据可依"的可靠感。

---

## 3. 核心价值观（Values）

### 第一：尊重传统命理

命理学有数千年的传承和体系。玄风门不做标新立异的"创新"，而是在传统框架内进行严谨的数字化实现。所有算法逻辑遵循经典命理原理，不编造、不简化。

### 第二：坚持可解释

任何分析结论必须说明来源和依据。用户有权知道：这个结论是怎么来的？基于什么规则？可信度如何？不可解释的结果等于不可信任。

### 第三：坚持真实案例驱动

不是基于想象优化产品，而是基于真实用户、真实案例、真实反馈持续改进。数据比直觉更可靠。案例比假设更有说服力。

### 第四：坚持长期主义

不追求短期数据暴涨。每一个产品决策都考虑三年后的影响。用户信任一旦失去，很难重建。宁可慢一点，也要稳一点。

### 第五：用户信任高于商业利益

商业变现是必要的，但不得损害用户信任。任何可能让用户产生"被欺骗"感觉的商业模式，都不应被采纳。短期收入不能凌驾于长期信任之上。

---

## 4. 产品边界（Boundary）

玄风门明确自身的能力边界，不越界、不误导。

**玄风门不是：**

- 不是预测一切的"算命机"
- 不是替用户做决定的"决策代理"
- 不是宣称"百发百中"的玄学工具
- 不是用神秘包装获取信任的营销产品

**玄风门是：**

- 帮助用户理解命局的分析工具
- 帮助用户看清趋势和方向的参考系统
- 辅助用户在重要节点做出更知情决定的伙伴

**最终决定权，永远属于用户。**

我们提供分析视角，但不代替用户思考和选择。

---

## 5. 长期成功标准

衡量玄风门项目是否成功，不只看短期收入数字。

以下指标同等重要：

| 指标 | 说明 |
|------|------|
| **用户满意度** | 综合满意度评分 >= 4.0/5.0 |
| **用户留存** | 7日/30日/90日留存率持续增长 |
| **反馈质量** | 高质量用户反馈 >= 100条 |
| **真实案例增长** | 案例库、知识库持续积累 |
| **专家认可度** | 命理专家对平台分析结果的专业认可 |
| **报告阅读完成率** | 用户愿意完整阅读报告的比例 |

收入是结果，不是目标。
当用户真正信任产品、持续使用产品、愿意推荐给他人时，收入自然随之而来。

> 长期价值 > 短期数字。

---

## 6. V6 启动原则

V6 不是按时间表启动的。

V6 的启动必须满足以下条件：

1. **真实数据证明**当前 V5 算法存在可以验证的提升空间
2. **用户反馈**中明确指向算法层面的系统性问题
3. **专家验证**确认了改进方向和具体方案
4. **数据资产**积累到足够支撑算法迭代的规模（案例库 >= 1000，知识库 >= 500，专家审核 >= 50）

**如果以上条件不满足，继续运营 V5。**

不因为"时间到了"而启动新版本。
不因为"别人做了"而盲目跟进。
只因为有真实的、可验证的、数据支撑的理由才启动。

## 文档体系

玄风门所有文档遵循以下层级结构：

```
PRODUCT_VISION.md        — 愿景（为什么存在）
        ↓
PRODUCT_CONSTITUTION.md  — 原则（怎么做事）
        ↓
ARCHITECTURE.md          — 架构（技术怎么实现）
        ↓
Professional Engine      — 引擎（算法核心）
        ↓
Web Product Layer        — 产品（用户界面）
```

| 层级 | 文档 | 回答的问题 |
|------|------|-----------|
| 愿景 | PRODUCT_VISION.md | 我们为什么存在？我们要成为什么？ |
| 原则 | PRODUCT_CONSTITUTION.md | 我们怎么做事？什么能做？什么不能做？ |
| 架构 | ARCHITECTURE.md | 技术系统如何组织？ |
| 实现 | Professional Engine | 命理算法如何计算？ |
