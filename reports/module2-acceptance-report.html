1.4 核心设计决策

  • 数据库驱动架构：所有38种神煞统一定义在 SHEN_SHA_DATABASE 数组中，Calculator 签名统一为 (8 gan/zhi + gender) => { hit, positions, confidence }
  • 零侵入集成：Module 2 引擎接收 ProfessionalFourPillarsResult 作为输入，不修改 Module 1 任何代码
  • 可配置过滤：支持按分类过滤、按ID过滤、冲突检测开关，全部通过 ShenShaEngineOptions 传入
  • 缓存机制：Map-based 内存缓存，相同输入二次计算直接返回，缓存键基于四柱干支+性别+过滤条件
  • AI Explain 结构化：不直接写死解释文本，输出 basis / classicalReference / modernInterpretation / conditions / conflictSituation / confidenceAssessment / suggestions 结构化数据

2.4 冲突检测机制

Module 2 实现了自动冲突检测：

  • 检测逻辑：遍历所有命中神煞，检查其 conflicts 数组是否与当前命中神煞 ID 存在交集
  • 已知冲突对：羊刃 ↔ 飞刃、孤辰 ↔ 寡宿、桃花 ↔ 灾煞等
  • 输出形式：触发 MULTI_RULE_CONFLICT 警告，并在冲突信息中给出 resolution 建议
  • 可关闭：detectConflicts: false 可跳过检测以提升性能

4.2 性能优化措施

  • Map-based 缓存：以四柱干支+性别+过滤条件为键，缓存完整输出对象
  • 惰性计算：未请求的分类/ID不进入计算循环
  • 批量推导：单次遍历完成过滤、计算、冲突检测、分组、排序
  • 无外部 I/O：全部纯内存计算，无数据库/网络调用
